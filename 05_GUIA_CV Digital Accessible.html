📏 Unitats relatives (rem / em)

Per què és important? Permet que l'usuari pugui augmentar o disminuir la mida del text des del navegador sense trencar el disseny.

Com funciona?

  • 1rem = mida base del navegador (normalment 16px)
  • 1.5rem = 24px
  • em és relatiu al pare, rem és relatiu a l'arrel.

Exemple:

body { font-size: 1rem; } /* 16px */
h1 { font-size: 2rem; } /* 32px */
.small { font-size: 0.875rem; } /* 14px */

🛠️ Codi CSS utilitzat

Aquesta plantilla utilitza una base de mides relatives:

:root { font-size: 16px; }
body { font-size: 1rem; }
h1 { font-size: 2.5rem; }
.profile-card { font-size: 0.95rem; }
.btn { font-size: 1.1rem; }

Consell: Evita valors fixos en píxels per als textos principals.

📊 Verificació de contrast

La WCAG 2.1 exigeix un contrast mínim de 4.5:1 per al text normal.

Eines recomanades:

  • WebAIM Contrast Checker
  • Colorable
  • Extensions de navegador (WAVE, axe).

Els nostres colors: #2c5aa0 (blau) sobre blanc → contrast 7.2:1 (✅ compleix).

🎨 Paleta accessible

Colors principals:

  • ■ Primari: #2c5aa0 (blau fosc)
  • ■ Secundari: #4a8fe7 (blau clar)
  • ■ Fons clar: #f8fafc
  • ■ Text: #212529 (gris fosc)

Tots els colors compleixen WCAG AA (contrast ≥4.5:1).

⌨️ Focus visible

El focus indica quin element està seleccionat quan es navega amb teclat (tabulador).

Característiques d'un bon focus:

  • Visible (color contrastat)
  • Prou gruixut per distingir-se
  • No molesta la lectura

A la plantilla: outline: 3px solid #ffbf47; outline-offset: 4px;

📝 :focus-visible explicat

:focus-visible és una pseudoclase moderna que només mostra el focus quan és realment necessari (teclat), no quan es fa clic amb ratolí.

Exemple:

/* Només per a navegació per teclat */
:focus-visible {
  outline: 3px solid #ffbf47;
  outline-offset: 4px;
}

🔤 Atribut alt

L'atribut alt proporciona un text alternatiu per a les imatges. És llegit pels lectors de pantalla.

Bones pràctiques:

  • Descriu el contingut de la imatge.
  • Si és decorativa, posa alt="" (buit).
  • No comencis amb "Imatge de...".

Exemple: <img src="avatar.jpg" alt="Maria Garcia, instal·ladora electricista">

📌 ARIA (Accessible Rich Internet Applications)

ARIA proporciona atributs addicionals per millorar l'accessibilitat quan HTML no és suficient.

Més utilitzats:

  • aria-label: etiqueta per a elements sense text visible.
  • aria-labelledby: relaciona amb un altre element.
  • aria-hidden: oculta elements decoratius.

A la plantilla: <button aria-label="Tancar">×</button>

📐 Jerarquia d'encapçalats

Els lectors de pantalla utilitzen els encapçalats (h1 a h6) per navegar per la pàgina.

Regles bàsiques:

  • Un sol h1 per pàgina.
  • No saltis nivells (h1 → h3 no és recomanable).
  • Han de descriure el contingut de la secció.

🧱 HTML semàntic

Utilitzar elements HTML pel seu significat, no només per la presentació.

Elements clau:

  • <header>, <nav>, <main>, <footer>
  • <article>, <section>, <aside>
  • <table> amb <caption> i <th>

Aquesta plantilla utilitza HTML semàntic en totes les seccions.

📱 Disseny responsive

Què és? La capacitat d'una pàgina web d'adaptar-se a diferents mides de pantalla (mòbil, tauleta, ordinador).

Com ho implementem?

  • Mòbil: Media query (max-width: 768px) · columnes apilades, text més gran
  • Tauleta: Media query (769px - 1024px) · graella de 2 columnes
  • Ordinador: Graella de 3-6 columnes segons secció

Exemple:

@media (max-width: 768px) {
  .profiles-grid { grid-template-columns: 1fr; }
}
@media (min-width: 769px) and (max-width: 1024px) {
  .profiles-grid { grid-template-columns: repeat(2, 1fr); }
}

📐 Breakpoints

Per què 768px i 1024px?

  • 768px és l'amplada típica d'un iPad en vertical i de molts mòbils en horitzontal.
  • 1024px és l'amplada típica d'un iPad en horitzontal.
  • Aquests valors cobreixen la majoria de dispositius del mercat.

📋 WCAG 2.1 (nivell AA)

WCAG = Web Content Accessibility Guidelines (Pautes d'Accessibilitat per al Contingut Web).

Nivell AA: És el nivell estàndard exigit a Europa. Compleix els 50 criteris de nivell A i AA.

Principis:

  • Percebible: La informació s'ha de poder percebre (contrast, alternatives textuals).
  • Operable: Interfície navegable (teclat, temps suficient).
  • Comprensible: Contingut llegible i previsible.
  • Robust: Compatible amb diferents tecnologies.
